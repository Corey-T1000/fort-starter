# scratch/research/

Investigations, deliverables, and findings. The output of `/research` and similar deep-dive skills lands here before being promoted into `memory/` (durable knowledge) or discarded.

## Conventions

- One subdir or markdown file per investigation: `research/<topic>.md` or `research/<topic>/`
- Include sources (links, file paths, dates) — research without sources rots fast
- When findings stabilize and are reusable, promote to a `memory/XX-topic.md` file
- Date the filename when the investigation is time-sensitive: `research/2026-04-22-vendor-comparison.md`

## Examples

```
research/
  framework-comparison.md
  2026-04-15-database-migration-options/
    findings.md
    benchmarks.csv
```
